Understanding Big O Notation

Big O notation describes the performance or complexity of an algorithm. It specifically describes the worst-case scenario and how the runtime or space requirements grow as the input size increases.

O(1)

Constant Time

Fastest. Time doesn't change with input size.

O(log n)

Logarithmic Time

Very efficient. Time grows slowly.

O(n)

Linear Time

Time grows directly with input size.

O(n²)

Quadratic Time

Slower. Time grows with square of input.

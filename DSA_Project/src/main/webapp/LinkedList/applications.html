Why Linked Lists Matter

Linked lists excel in scenarios requiring dynamic memory allocation and frequent structural modifications.

Dynamic Size

No need to pre-allocate memory. Grows and shrinks as needed during runtime.

Fast Insertions

O(1) insertion/deletion time when you have a pointer to the insertion point.

Memory Efficient

Only uses memory for actual elements, no wasted space for unused capacity.

Non-Contiguous

Nodes can be scattered in memory, reducing fragmentation issues.

Code Examples

See how linked lists are implemented in real-world scenarios.

1. Singly Linked List Implementation

class Node {
    constructor(data) {
        this.data = data;
        this.next = null;
    }
}

class LinkedList {
    constructor() {
        this.head = null;
    }
    
    insertAtEnd(data) {
        const node = new Node(data);
        if (!this.head) {
            this.head = node;
            return;
        }
        let current = this.head;
        while (current.next) {
            current = current.next;
        }
        current.next = node;
    }
}

2. Doubly Linked List

class DoublyNode {
    constructor(data) {
        this.data = data;
        this.next = null;
        this.prev = null;
    }
}

class DoublyLinkedList {
    insertAfter(refNode, data) {
        const newNode = new DoublyNode(data);
        newNode.next = refNode.next;
        newNode.prev = refNode;
        if (refNode.next) {
            refNode.next.prev = newNode;
        }
        refNode.next = newNode;
    }
}

3. Circular Linked List

class CircularLinkedList {
    constructor() {
        this.head = null;
    }
    
    insert(data) {
        const newNode = new Node(data);
        if (!this.head) {
            this.head = newNode;
            newNode.next = newNode;
        } else {
            let current = this.head;
            while (current.next !== this.head) {
                current = current.next;
            }
            current.next = newNode;
            newNode.next = this.head;
        }
    }
}

Wrong vs Right Queue Implementation

See the difference between common mistakes and correct queue solutions

❌ Common Mistake

// Linear queue - inefficient shifting class Queue { constructor() { this.items = []; } enqueue(item) { this.items.push(item); } dequeue() { return this.items.shift(); } // O(n) } // No empty check queue.dequeue(); // Runtime error

Shift operation is O(n) and no safety checks.

✅ Correct Approach

// Circular queue - O(1) operations class Queue { constructor(size) { this.size = size; this.items = new Array(size); this.front = -1; this.rear = -1; } enqueue(item) { if (this.isFull()) return false; if (this.isEmpty()) this.front = 0; this.rear = (this.rear + 1) % this.size; this.items[this.rear] = item; return true; } dequeue() { if (this.isEmpty()) return null; const item = this.items[this.front]; this.front = (this.front + 1) % this.size; return item; } }

O(1) operations with proper bounds checking and circular implementation.

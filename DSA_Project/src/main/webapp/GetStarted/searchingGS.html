What are Searching Algorithms?

Searching algorithms find the position of a target value within a sorted/unsorted array. Linear Search works on any array (O(n)), while Binary Search requires sorted arrays (O(log n)).

What are Searching Algorithms?

Searching algorithms locate the position of a target value within a collection, such as an array or list. They are fundamental operations in computer science with applications in databases, file systems, and data processing.

The choice between Linear Search (O(n)) and Binary Search (O(log n)) depends on whether the data is sorted and performance requirements.

Types of Searching Algorithms

  • Linear Search: Sequential search through unsorted data (O(n)).
  • Binary Search: Divide-and-conquer on sorted data (O(log n)).
  • Jump Search: Block-wise search jumping ahead (O(√n)).
  • Interpolation Search: Enhanced binary search using interpolation (O(log log n)).
  • Exponential Search: Finds range for binary search (O(log n)).

Key Concepts

  • Sorted vs Unsorted: Binary search requires sorted data.
  • Return Values: Index of found element or -1 (not found).
  • Time Complexity: Critical for large datasets.
  • Space Complexity: Usually O(1) for iterative implementations.

Advantages

  • Simple to implement (especially linear search).
  • Binary search is extremely efficient for large sorted datasets.
  • No additional data structures needed.

Limitations

  • Binary search requires sorted data (O(n log n) preprocessing).
  • Not suitable for frequently changing data.

Linear Search

Basic Implementation

int linearSearch(int arr[], int n, int key) {
    for(int i = 0; i < n; i++)
        if(arr[i] == key) return i;
    return -1;
}  // C

Sentinel Linear Search

int sentinelSearch(int arr[], int n, int key) {
    arr[n] = key;
    int i = 0;
    while(arr[i] != key) i++;
    return (i < n) ? i : -1;
}  // C

Binary Search

Iterative

int binarySearch(int arr[], int n, int key) {
    int left = 0, right = n-1;
    while(left <= right) {
        int mid = left + (right - left) / 2;
        if(arr[mid] == key) return mid;
        if(arr[mid] < key) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}  // C

Recursive

int binarySearchRec(int arr[], int left, int right, int key) {
    if(left > right) return -1;
    int mid = left + (right - left) / 2;
    if(arr[mid] == key) return mid;
    if(arr[mid] > key) return binarySearchRec(arr, left, mid-1, key);
    return binarySearchRec(arr, mid+1, right, key);
}  // C

Jump Search

Basic Implementation

int jumpSearch(int arr[], int n, int key) {
    int step = sqrt(n);
    int prev = 0;
    while(arr[prev] < key) {
        prev = step;
        step += sqrt(n);
        if(prev >= n) break;
    }
    for(int i = prev; i < n && arr[i] <= key; i++)
        if(arr[i] == key) return i;
    return -1;
}  // C

Step-by-Step Algorithms

Detailed algorithms for the most important searching techniques.

Linear Search Algorithm

  1. Start from the first element (index 0).
  2. Compare target with current element.
  3. If match found, return current index.
  4. Move to next element.
  5. Repeat until end of array or match found.
  6. Return -1 if not found.
// Pseudocode
for i from 0 to n-1:
    if arr[i] == target:
        return i
return -1

Binary Search Algorithm (Iterative)

  1. Set left = 0, right = n-1.
  2. While left ≤ right:
  3. Calculate mid = left + (right - left) / 2.
  4. If arr[mid] == target, return mid.
  5. If arr[mid] < target, set left = mid + 1.
  6. Else, set right = mid - 1.
  7. Return -1 if loop ends.
// Pseudocode
left = 0, right = n-1
while left <= right:
    mid = left + (right - left) / 2
    if arr[mid] == target: return mid
    if arr[mid] < target: left = mid + 1
    else: right = mid - 1
return -1

Jump Search Algorithm

  1. Calculate step = √n.
  2. Jump forward by step size until arr[prev] ≥ target.
  3. Perform linear search in the block [prev, min(n, prev+step)].
  4. Return index if found, else -1.
// Pseudocode
step = sqrt(n)
prev = 0
while arr[prev] < target:
    prev = step
    step += sqrt(n)
    if prev >= n: break
// Linear search in block
for i from prev to min(n, prev+step):
    if arr[i] == target: return i
return -1

Real-World Applications

Searching algorithms power countless systems in modern computing.

  • Databases: SQL queries, index lookups, B-tree searches.
  • File Systems: Finding files by name, metadata search.
  • Search Engines: Inverted indexes, autocomplete (binary search on sorted results).
  • Compilers: Symbol tables, identifier lookup.
  • Networking: Routing tables, IP lookup.
  • Operating Systems: Process scheduling, memory management.
  • Games: Collision detection, AI pathfinding.
  • E-commerce: Product search, price filtering.
  • Machine Learning: Nearest neighbor search, k-d trees.
  • Graphics: Ray tracing, spatial partitioning.

Common Mistakes & Best Practices

Avoid these pitfalls when implementing searching algorithms.

  • Using Binary Search on Unsorted Data: Always verify array is sorted first.
  • Integer Overflow in mid: Use mid = left + (right - left)/2 instead of (left+right)/2.
  • Infinite Loop in Binary Search: Ensure left <= right condition.
  • Missing Edge Cases: Test empty arrays, single elements, target at boundaries.
  • Recursive Stack Overflow: Use iterative binary search for large arrays.
  • Not Handling Duplicates: Binary search finds first occurrence; specify requirements.
  • Forgetting Array Bounds: Validate indices before access.
  • Performance Misjudgment: Linear search better for small arrays (<100 elements).

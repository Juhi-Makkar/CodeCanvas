What is Sorting?

Sorting is the process of arranging elements in a specific order (ascending/descending). Essential for efficient searching, data analysis, and preprocessing.

What is Sorting?

Sorting algorithms arrange elements of an array or list in a specific order (ascending or descending). They are fundamental for efficient searching, data processing, and optimization problems. Different algorithms have different trade-offs in time/space complexity and stability.

For example, sorting [64, 34, 25, 12, 22, 11] → [11, 12, 22, 25, 34, 64].

Types of Sorting Algorithms

  • Comparison-based: Bubble, Selection, Insertion, Merge, Quick, Heap sort
  • Non-comparison: Counting, Radix, Bucket sort
  • Stable: Maintains relative order of equal elements (Merge, Insertion)
  • In-place: Requires O(1) extra space (Quick, Heap)
  • Adaptive: Faster on nearly sorted data (Insertion)

Key Properties

  • Time Complexity: Best/Average/Worst case analysis
  • Space Complexity: Auxiliary space required
  • Stability: Preserves relative order of equal elements
  • In-place: Requires O(1) extra space (Quick, Heap)
  • Adaptive: Faster on nearly sorted data (Insertion)
  • Online: Can process elements as they arrive (Insertion)

Advantages of Sorting

  • Enables efficient searching (Binary search O(log n))
  • Simplifies many algorithms (Merge, closest pair)
  • Essential for data analysis and visualization
  • Prepares data for divide-and-conquer strategies

Limitations

  • General comparison sorts have Ω(n log n) lower bound
  • Some algorithms unstable or have worst-case O(n²)
  • Extra space required for some efficient algorithms

Selection Sort

Implementation

for(int i = 0; i < n-1; i++) {
    int min_idx = i;
    for(int j = i+1; j < n; j++)
        if(arr[j] < arr[min_idx])
            min_idx = j;
    swap(arr[i], arr[min_idx]);
}

Insertion Sort

Implementation

for(int i = 1; i < n; i++) {
    int key = arr[i];
    int j = i - 1;
    while(j >= 0 && arr[j] > key) {
        arr[j+1] = arr[j];
        j--;
    }
    arr[j+1] = key;
}

Merge Sort (Recursive)

Implementation

void mergeSort(int arr[], int l, int r) {
    if(l < r) {
        int m = l + (r-l)/2;
        mergeSort(arr, l, m);
        mergeSort(arr, m+1, r);
        merge(arr, l, m, r);
    }
}

Quick Sort

Implementation

int partition(int arr[], int low, int high) {
    int pivot = arr[high];
    int i = low - 1;
    for(int j = low; j < high; j++)
        if(arr[j] <= pivot) swap(arr[++i], arr[j]);
    swap(arr[i+1], arr[high]);
    return i+1;
}

Step-by-Step Algorithms

Bubble Sort Algorithm

  1. Repeat for n-1 passes
  2. In each pass, compare adjacent elements
  3. If out of order, swap them
  4. Reduce comparison range by 1 each pass
  5. Largest element bubbles to end each pass
// Pseudocode
for i from 0 to n-2:
    for j from 0 to n-i-2:
        if arr[j] > arr[j+1]:
            swap arr[j], arr[j+1]

Selection Sort Algorithm

  1. For each position i from 0 to n-2
  2. Find minimum element in unsorted portion
  3. Swap minimum with first unsorted element
  4. Unsored portion shrinks by 1
// Pseudocode
for i from 0 to n-2:
    min_idx = i
    for j from i+1 to n-1:
        if arr[j] < arr[min_idx]:
            min_idx = j
    swap arr[i], arr[min_idx]

Quick Sort (Partition Scheme)

  1. Choose pivot (last, first, random, median)
  2. Partition: smaller elements left, larger right
  3. Recursively sort left and right subarrays
  4. Base case: single element or empty
// Pseudocode
partition(arr, low, high):
    pivot = arr[high]
    i = low - 1
    for j from low to high-1:
        if arr[j] <= pivot:
            i++
            swap arr[i], arr[j]
    swap arr[i+1], arr[high]
    return i+1

Applications of Sorting

  • Searching: Binary search requires sorted arrays
  • Databases: ORDER BY clauses, index optimization
  • Data Analysis: Sorting for ranking, percentiles
  • Graph Algorithms: Dijkstra, Prim's (priority queues)
  • Closest Pair: Divide and conquer geometric problems
  • Merge Operations: External sorting, multi-way merges
  • Dynamic Programming: LIS, weighted interval scheduling
  • Computational Geometry: Convex hull, line segment intersection
  • Numerical Methods: Matrix operations, eigenvalue problems
  • Machine Learning: Feature ranking, k-NN preprocessing

Common Mistakes & Best Practices

  • Quick Sort Worst Case: Use random pivot or median-of-three
  • Infinite Recursion: Always check base cases in recursive sorts
  • Stack Overflow: Merge/Quick sort recursion depth O(log n)
  • Off-by-One: Loop bounds in bubble/selection sort
  • Unstable Sorting: Use stable sort when relative order matters
  • Array Bounds: Check indices before access
  • Pivot Selection: Poor pivot → worst case performance
  • Space Issues: Merge sort needs O(n) space
  • Small Arrays: Use Insertion sort for n < 10
  • Duplicates: Handle equal elements properly

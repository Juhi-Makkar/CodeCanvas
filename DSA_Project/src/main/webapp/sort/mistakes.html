Wrong vs Right Approach

See the difference between common mistakes and correct sorting implementations

❌ Common Mistake

// Bubble Sort - No early termination for (let i = 0; i < arr.length; i++) { for (let j = 0; j < arr.length; j++) { if (arr[j] > arr[j + 1]) { [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; } } } // Wrong comparator arr.sort((a, b) => a - b); // Fails with large numbers

Always swaps and no optimization - O(n²) even for sorted arrays.

✅ Correct Approach

// Optimized Bubble Sort with early termination let swapped; for (let i = 0; i < arr.length; i++) { swapped = false; for (let j = 0; j < arr.length - 1 - i; j++) { if (arr[j] > arr[j + 1]) { [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; swapped = true; } } if (!swapped) break; } // Safe numeric comparator arr.sort((a, b) => Number(a) - Number(b));

Early termination and optimized passes - O(n) for sorted arrays.
